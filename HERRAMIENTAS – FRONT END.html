HERRAMIENTAS

Cuando hablamos de tecnologías Front-End y del uso de frameworks, no basta con saber HTML, CSS y JavaScript; también necesitamos un conjunto de herramientas de apoyo que facilitan el trabajo del desarrollador. Estas se dividen principalmente en: editores de código, sistemas de control de versiones y automatizadores de tareas.

EDITORES DE CODIGO

  • Visual Studio Code (VS Code): el más popular, multiplataforma, con miles de extensiones.
  • Sublime Text: ligero, rápido y muy personalizable.
  • Atom: desarrollado por GitHub, aunque hoy en día menos usado.
  • WebStorm: especializado en JavaScript y frameworks como React, Angular y Vue.
  • Brackets: pensado para diseño web, con previsualización en vivo.

Sistemas de control de versiones:

Permiten llevar un historial del código, trabajar en equipo y volver a versiones anteriores en caso de errores.

  • Git: el más utilizado en el mundo, administra cambios en el código.
  • GitHub: plataforma en la nube para alojar proyectos Git y colaborar en equipo.
  • GitLab: alternativa a GitHub, con más enfoque en integración continua.
  • Bitbucket: otra opción popular, muy usada en entornos corporativos.

Automatizadores y gestores de tareas:

Son herramientas que facilitan tareas repetitivas como compilar código, optimizar imágenes, minificar archivos CSS/JS o recargar el navegador automáticamente.

  • Gulp: automatizador de tareas basado en JavaScript.
  • Grunt: uno de los primeros automatizadores, aunque hoy menos usado.
  • Webpack: empaquetador muy potente, organiza el proyecto en módulos y optimiza el rendimiento.
  • Parcel: alternativa más simple y rápida a Webpack.
  • Vite: herramienta moderna para proyectos con frameworks como Vue, React o Svelte.
  • NPM (Node Package Manager): gestor de paquetes que también puede ejecutar scripts para automatizar procesos.
  • Yarn / PNPM: alternativas a NPM, más rápidas y eficientes.
